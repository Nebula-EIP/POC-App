# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public issue.

Instead, report it privately to the maintainers through your designated private channel.

Include:

- A clear description of the issue
- Steps to reproduce
- Potential impact
- Suggested mitigation (if available)

## Response Expectations

- We will acknowledge receipt as soon as possible.
- We will investigate and triage the report.
- We will coordinate a fix and disclosure timeline when needed.

## Scope

This policy applies to source code, workflows, scripts, and documentation in this repository.
